## 実装詳細

### 1. WebGLビルド設定 🔵

**信頼性**: 🔵 *tech-stack.md、REQ-603より*

Unity Player SettingsでWebGLビルドの設定を行う。

- **Platform**: WebGL
- **Compression Format**: Brotli（推奨。サーバーが対応しない場合はGzip）
- **Color Space**: Linear or Gamma（プロジェクト設定に合わせる）
- **Exception Handling**: Explicitly Thrown Exceptions Only（ビルドサイズ削減のため）
- **Data Caching**: 有効
- **Run In Background**: 無効（ブラウザタブ非アクティブ時の挙動制御）

### 2. 最適化 🔵

**信頼性**: 🔵 *NFR-004より*

ビルドサイズと実行パフォーマンスの最適化設定。

- **Sprite Atlas有効化**: TASK-0002で作成済みのSprite Atlasを確認
- **Strip Engine Code**: 有効（未使用のUnity Engineコードを除外）
- **Managed Stripping Level**: Medium or High（IL2CPP環境でのコード削減）
- **テクスチャ圧縮設定**: WebGL向け最適化（ASTC or ETC2）
- **Audio Compression**: Vorbis（品質設定の最適化）
- **未使用アセットの除外**: Build Reportで確認し不要アセットを削除

### 3. ビルドサイズ確認 🔵

**信頼性**: 🔵 *NFR-004より*

ビルド成果物のサイズを確認し、目標値以内であることを検証する。

- **目標**: 初期ロード30MB以下
- **Build Report**: Editor.logからビルドレポートを確認
- **各アセットカテゴリのサイズ**: テクスチャ、オーディオ、スクリプト、シェーダー
- サイズ超過の場合: テクスチャ解像度の調整、オーディオ品質の調整、不要アセットの除外

### 4. ブラウザ動作確認 🔵

**信頼性**: 🔵 *NFR-201より*

主要ブラウザでの動作確認を行う。

- **Chrome最新版**: WebGL 2.0対応確認、60fps動作確認
- **Firefox最新版**: WebGL 2.0対応確認、60fps動作確認
- **Edge最新版**: WebGL 2.0対応確認、60fps動作確認
- **確認項目**:
  - タイトル画面の表示・操作
  - AI人数選択・ゲーム開始
  - カード出し・パス操作
  - AI思考・アクション実行
  - 勝利判定・結果表示
  - タイトルに戻る
  - 60fps安定動作（NFR-001）

## 実装手順

### DIRECT手順

1. **direct-setup**: Unity Build SettingsでWebGLプラットフォームに切り替える
2. **direct-setup**: Player Settingsで Compression Format: Brotli、Exception Handling: Explicitly Thrown Exceptions Only を設定する
3. **direct-setup**: Strip Engine Code: 有効、Managed Stripping Level: Medium を設定する
4. **direct-setup**: テクスチャ圧縮設定をWebGL向けに最適化する
5. **direct-setup**: Sprite Atlasが有効であることを確認する
6. **direct-verify**: WebGLビルドを実行し、ビルドが成功することを確認する
7. **direct-verify**: ビルド成果物のサイズを確認し、30MB以下であることを検証する
8. **direct-verify**: ローカルサーバー（Python http.server等）でビルドをホストする
9. **direct-verify**: Chromeで動作確認を行う（全ゲームフロー + 60fps確認）
10. **direct-verify**: Firefoxで動作確認を行う（全ゲームフロー + 60fps確認）
11. **direct-verify**: Edgeで動作確認を行う（全ゲームフロー + 60fps確認）
12. **direct-verify**: サイズ超過またはパフォーマンス問題がある場合、最適化設定を調整して再ビルドする
