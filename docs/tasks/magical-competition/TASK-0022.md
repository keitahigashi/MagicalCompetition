## 実装詳細

### 1. WaitForInput() 🔵

**信頼性**: 🔵 *dataflow.md UI操作フローより*

プレイヤーのカード選択・確定操作を待ち受ける。

```csharp
public class PlayerInputController
{
    public void WaitForInput()
    public void SelectCard(Card card)
    public void DeselectCard(Card card)
    public bool CanConfirmPlay()
    public PlayAction ConfirmPlay()
    public void StartPassSelection()
}
```

- カード選択状態の管理: 選択されたカードのリストを保持（複数選択可能）
- 選択カードのリアルタイム妥当性チェック: 選択が変わるたびにPlayValidatorで合法性を検証
- 「出す」ボタン: CanConfirmPlay()がtrueの場合のみ活性化
- 「パス」ボタン: StartPassSelection()でパスフローへ移行

**実装ファイル**: `Assets/_Project/Scripts/Controllers/PlayerInputController.cs`

### 2. SelectCardsToReturn() 🔵

**信頼性**: 🔵 *dataflow.md パスフローより*

パス時に山札に戻すカードを選択する。

```csharp
public void SelectCardToReturn(Card card)
public void DeselectCardToReturn(Card card)
public PlayAction ConfirmPass()
```

- パス時に山札に戻すカード選択（0枚も可） (REQ-106)
- 全手札を戻すことも可能 (EDGE-104)
- 「確定」でPlayAction.CreatePass()で返す

**実装ファイル**: `Assets/_Project/Scripts/Controllers/PlayerInputController.cs`

### 3. イベント 🔵

**信頼性**: 🔵 *dataflow.md イベントフローより*

```csharp
public event Action<Card> OnCardSelected;
public event Action<Card> OnCardDeselected;
public event Action<PlayAction> OnPlayConfirmed;
public event Action<PlayAction> OnPassConfirmed;
public event Action<bool> OnPlayValidityChanged;
```

- `OnCardSelected`: カード選択時に発火
- `OnCardDeselected`: カード選択解除時に発火
- `OnPlayConfirmed`: カード出し確定時に発火
- `OnPassConfirmed`: パス確定時に発火
- `OnPlayValidityChanged`: 妥当性チェック結果が変化した時に発火

**実装ファイル**: `Assets/_Project/Scripts/Controllers/PlayerInputController.cs`

## 単体テスト要件

### 1. カード選択: 選択状態が切り替わる 🔵

SelectCardでカードが選択状態になり、DeselectCardで選択解除されることを確認する。

### 2. 複数選択: 複数カードが選択される 🔵

複数のカードを選択でき、全て選択状態として保持されることを確認する。

### 3. 妥当性チェック: 有効な組み合わせで「出す」ボタン活性化 🔵

PlayValidatorが合法と判定する組み合わせを選択した場合、CanConfirmPlay()がtrueを返すことを確認する。

### 4. 妥当性チェック: 無効な組み合わせで「出す」ボタン非活性 🔵

PlayValidatorが非合法と判定する組み合わせを選択した場合、CanConfirmPlay()がfalseを返すことを確認する。

### 5. パス: 戻すカード0枚でもOK 🔵 (EDGE-104)

パス選択時に戻すカードを0枚に設定してもConfirmPass()が正常にPlayActionを返すことを確認する。

### 6. パス: 全手札を戻すことも可能 🔵 (EDGE-104)

パス選択時に全手札を戻すカードとして選択してもConfirmPass()が正常にPlayActionを返すことを確認する。

**テストファイル**: `Assets/_Project/Tests/EditMode/Controllers/PlayerInputControllerTests.cs`

## 実装手順

### TDD手順

1. **tdd-requirements**: dataflow.mdからUI操作フロー・パスフローの仕様を確認する
2. **tdd-testcases**: 上記6つのテストケースをPlayerInputControllerTests.csに記述する
3. **tdd-red**: テストを実行し、全てREDであることを確認する
4. **tdd-green**: PlayerInputControllerクラスを実装し、全テストをGREENにする

   - CanConfirmPlay: PlayValidatorに選択カードと場札を渡して合法性を判定
   - ConfirmPlay: PlayAction.CreatePlayで選択カードからPlayActionを生成
   - StartPassSelection/ConfirmPass: パスモードへの切り替え、PlayAction.CreatePassで生成
5. **tdd-refactor**: 必要に応じてリファクタリングする（状態管理のState Pattern化、イベント整理など）
6. **tdd-verify-complete**: 全テストがパスし、完了条件を満たすことを確認する
