## 実装詳細

### 1. DecideAction 🔵

**信頼性**: 🔵 *interfaces.md NormalStrategyより*

全有効手から最適な手を選択する。選択基準は以下の通り:

```csharp
public class NormalStrategy : IAIStrategy
{
    public PlayAction DecideAction(GameState state, List<PlayAction> validPlays)
}
```

- 出せるカードがあれば、最も多く出せる手（枚数が多い手）を優先
- 同枚数の場合は、同数字 > 同色 > 計算出し の優先順位で選択
- 出せるカードがない場合はパスを返す
- パス時は次のターンで使いにくいカード（場札と数字・色が離れているカード）を山札に戻す

**実装ファイル**: `Assets/_Project/Scripts/Core/AI/NormalStrategy.cs`

### 2. SelectFieldColor 🔵

**信頼性**: 🔵 *interfaces.md NormalStrategyより*

場札リセット時に選択する色を決定する。

```csharp
public CardColor SelectFieldColor(GameState state)
```

- 手札に最も多い色を選択する
- 同数の色がある場合はいずれかを返す（決定的な挙動）

**実装ファイル**: `Assets/_Project/Scripts/Core/AI/NormalStrategy.cs`

## 単体テスト要件

### 1. DecideAction(出せる手あり): カードプレイを返す 🔵

有効手が存在する場合、パスではなくカードプレイのPlayActionが返されることを確認する。

### 2. DecideAction(複数手): 枚数が多い手を優先 🔵

複数の有効手がある場合、出せる枚数が最も多い手が選択されることを確認する。

### 3. DecideAction(同枚数): 同数字>同色>計算出しの優先順位 🔵

同枚数の有効手が複数ある場合、同数字マッチが同色マッチより優先され、同色マッチが計算出しより優先されることを確認する。

### 4. DecideAction(出せない): パスを返す 🔵

有効手が空リストの場合、パスのPlayActionが返されることを確認する。

### 5. DecideAction(パス時): 不要カードを山札に戻す 🔵

パス選択時に、次のターンで使いにくいカードが山札に戻すカードとして設定されることを確認する。

### 6. SelectFieldColor: 手札に最も多い色を返す 🔵

手札の色分布を見て、最も枚数が多い色が返されることを確認する。

### 7. SelectFieldColor(同数): いずれかの色を返す 🔵

複数の色が同数の場合、いずれかの有効な色が返されることを確認する（nullや不正値でないこと）。

**テストファイル**: `Assets/_Project/Tests/EditMode/Core/AI/NormalStrategyTests.cs`
