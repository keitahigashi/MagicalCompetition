## タスク概要

場札と同じ数字・同じ色でカードを出せるかの判定ロジックを実装する。PlayValidatorクラスの基盤として、同数字プレイと同色プレイの判定メソッドを作成し、統合的なValidateメソッドを部分実装する（計算出し判定はTASK-0012で追加）。

- **前提タスク**: [TASK-0006](TASK-0006.md) (FieldState クラス)
- **後続タスク**: [TASK-0012](TASK-0012.md) (PlayValidator - 計算出し・全有効手探索)

## 完了条件

- [ ] IsSameNumberPlayが場札と同じ数字のカード群を正しく判定できる
- [ ] IsSameColorPlayが場札と同じ色のカード群を正しく判定できる
- [ ] Validateが同数字・同色いずれかに合致する場合にtrue/PlayTypeを返す
- [ ] 疑似場札（IsVirtual=true）の場合も正しく判定できる
- [ ] 全テストケース（8件）がグリーンである

## 実装詳細

### 1. IsSameNumberPlay(List\<Card\> cards, FieldState field) 🔵

**信頼性**: 🔵 *interfaces.md PlayValidator、REQ-101より*

選択された全カードの番号が場札の番号と一致するかを判定する。

```csharp
public bool IsSameNumberPlay(List<Card> cards, FieldState field)
```

- 1〜3枚のカード全ての `Number` が `field.Number` と一致すれば true
- カードの色は問わない

**実装ファイル**: `Assets/_Project/Scripts/Core/Systems/PlayValidator.cs`

### 2. IsSameColorPlay(List\<Card\> cards, FieldState field) 🔵

**信頼性**: 🔵 *interfaces.md PlayValidator、REQ-102より*

選択された全カードの色が場札の色と一致するかを判定する。

```csharp
public bool IsSameColorPlay(List<Card> cards, FieldState field)
```

- 1〜3枚のカード全ての `Color` が `field.Color` と一致すれば true
- カードの番号は問わない
- 疑似場札（`field.Color == CardColor.Any`）の場合、全カードが同じ色であれば true

**実装ファイル**: `Assets/_Project/Scripts/Core/Systems/PlayValidator.cs`

### 3. Validate(List\<Card\> selectedCards, FieldState field) 🔵

**信頼性**: 🔵 *interfaces.md PlayValidatorより*

いずれかのプレイタイプに合致するかを判定し、結果を返す。本タスクでは同数字・同色のみ判定し、計算出しはTASK-0012で追加する。

```csharp
public PlayValidateResult Validate(List<Card> selectedCards, FieldState field)
// PlayValidateResult { bool IsValid, PlayType Type }
```

**実装ファイル**: `Assets/_Project/Scripts/Core/Systems/PlayValidator.cs`

## 単体テスト要件

### 1. 同数字1枚: 場札Fire:5, [Water:5] → valid 🔵

場札と同じ番号5のカード1枚が有効と判定されることを確認する。

### 2. 同数字2枚: 場札Fire:5, [Water:5, Earth:5] → valid 🔵

場札と同じ番号5のカード2枚がまとめて有効と判定されることを確認する。

### 3. 同数字3枚: 場札Fire:5, [Water:5, Earth:5, Light:5] → valid 🔵

場札と同じ番号5のカード3枚がまとめて有効と判定されることを確認する。

### 4. 同数字不一致: 場札Fire:5, [Water:3] → invalid 🔵

場札と異なる番号のカードが無効と判定されることを確認する。

### 5. 同色1枚: 場札Fire:5, [Fire:3] → valid 🔵

場札と同じ色Fireのカード1枚が有効と判定されることを確認する。

### 6. 同色2枚: 場札Fire:5, [Fire:3, Fire:7] → valid 🔵

場札と同じ色Fireのカード2枚がまとめて有効と判定されることを確認する。

### 7. 同色混在: 場札Fire:5, [Fire:3, Water:7] → invalid (同色でない) 🔵

異なる色が混在するカード群が無効と判定されることを確認する。

### 8. 疑似場札(Any): FieldState.IsVirtual=true, number=5の場合も正しく判定 🔵

疑似場札の場合、同数字判定は番号5で行い、同色判定はカード群の色統一のみをチェックすることを確認する。

**テストファイル**: `Assets/_Project/Tests/EditMode/Core/Systems/PlayValidatorTests.cs`

## 実装手順

### TDD手順

1. **tdd-requirements**: interfaces.mdからPlayValidatorの仕様、REQ-101, REQ-102のルールを確認する
2. **tdd-testcases**: 上記8つのテストケースをPlayValidatorTests.csに記述する
3. **tdd-red**: テストを実行し、全てREDであることを確認する
4. **tdd-green**: PlayValidatorクラスを実装し、全テストをGREENにする

   - IsSameColorPlay: field.IsVirtualの場合はカード群の色統一チェック、通常時はfield.Colorとの一致チェック
   - Validate: IsSameNumberPlay → IsSameColorPlay の順で判定
5. **tdd-refactor**: Validateの戻り値型（PlayValidateResult）の設計を整理する
6. **tdd-verify-complete**: 全テストがパスし、完了条件を満たすことを確認する
