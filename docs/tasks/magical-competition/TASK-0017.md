## 実装詳細

### 1. IAIStrategy interface 🔵

**信頼性**: 🔵 *interfaces.md IAIStrategyより*

AI戦略の共通インターフェースを定義する。各難易度（Easy/Normal/Hard）はこのインターフェースを実装する。

```csharp
public interface IAIStrategy
{
    PlayAction DecideAction(GameState state, List<PlayAction> validPlays);
    CardColor SelectFieldColor(GameState state);
}
```

- `DecideAction`: 全有効手のリストから最適な手を選択して返す
- `SelectFieldColor`: 場札リセット時の色選択を行う

**実装ファイル**: `Assets/_Project/Scripts/Core/AI/IAIStrategy.cs`

### 2. AIActionEvaluator class 🔵

**信頼性**: 🔵 *architecture.md Core/AIより*

PlayValidatorを使用して、現在の手札と場札の状態から出せる全ての有効手を探索するクラス。

```csharp
public class AIActionEvaluator
{
    public List<PlayAction> FindAllValidPlays(List<Card> hand, FieldState field)
}
```

- 同数字マッチ: 場札と同じ数字のカードを手札から探索
- 同色マッチ: 場札と同じ色のカードを手札から探索
- 計算出し（2〜3枚）: 手札の2〜3枚の組み合わせで場札の数字になる組み合わせを全探索
- EDGE-401: 全探索により有効手の漏れがないことを保証する

**実装ファイル**: `Assets/_Project/Scripts/Core/AI/AIActionEvaluator.cs`

## 単体テスト要件

### 1. FindAllValidPlays: 同数字の有効手が含まれる 🔵

手札に場札と同じ数字のカードがある場合、その有効手がリストに含まれることを確認する。

### 2. FindAllValidPlays: 同色の有効手が含まれる 🔵

手札に場札と同じ色のカードがある場合、その有効手がリストに含まれることを確認する。

### 3. FindAllValidPlays: 計算出しの有効手が含まれる 🔵

手札の2〜3枚の組み合わせで場札の数字になる計算出しの有効手がリストに含まれることを確認する。

### 4. FindAllValidPlays: 出せるカードがない場合は空リスト 🔵

手札に有効な手がない場合、空のリストが返されることを確認する。

### 5. FindAllValidPlays: 全探索で漏れなく列挙される 🔵 (EDGE-401)

複数の有効手が存在する場面で、全ての有効手が漏れなく列挙されることを確認する。同数字・同色・計算出しの全パターンを網羅する。

**テストファイル**: `Assets/_Project/Tests/EditMode/Core/AI/AIActionEvaluatorTests.cs`
