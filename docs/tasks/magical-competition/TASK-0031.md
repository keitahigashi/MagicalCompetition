## 実装詳細

### 1. GameUI 🔵

**信頼性**: 🔵 *architecture.md GameUIより*

ゲーム画面の操作パネルコンポーネント。「出す」「パス」「確定」ボタンとパス時カード戻し選択UIを管理する。

```csharp
public class GameUI : MonoBehaviour
{
    [SerializeField] private Button _playButton;    // 「出す」ボタン
    [SerializeField] private Button _passButton;    // 「パス」ボタン
    [SerializeField] private Button _confirmButton; // 「確定」ボタン（パス時カード戻し確定用）
    [SerializeField] private GameObject _returnCardPanel; // パス時カード戻し選択パネル

    public event Action OnPlayButtonClicked;
    public event Action OnPassButtonClicked;
    public event Action OnReturnConfirmClicked;
}
```

- 「出す」ボタン: 有効な選択がある場合のみ活性化
- 「パス」ボタン: 常に活性化、カード出せない場合も使用可能
- パス時のカード戻し選択UI
- 「確定」ボタン: パス時のカード戻し確定用

**実装ファイル**: `Assets/_Project/Scripts/UI/GameUI.cs`

### 2. ボタン状態管理 🔵

**信頼性**: 🔵 *NFR-101より*

各ボタンの活性化/非活性化を制御するメソッド群。

```csharp
public void SetPlayButtonEnabled(bool enabled)
{
    // 「出す」ボタンの活性化制御
    _playButton.interactable = enabled;
}

public void SetPassButtonEnabled(bool enabled)
{
    // 「パス」ボタンの活性化制御
    _passButton.interactable = enabled;
}

public void ShowReturnCardSelection()
{
    // パス時カード戻し選択UIを表示
    _returnCardPanel.SetActive(true);
}

public void HideReturnCardSelection()
{
    // パス時カード戻し選択UIを非表示
    _returnCardPanel.SetActive(false);
}
```

### 3. PlayerInputControllerとの連携 🔵

**信頼性**: 🔵 *architecture.md Controllersより*

PlayerInputControllerがGameUIのイベントを購読し、プレイヤー操作を処理する。

- OnPlayButtonClicked: カード出し操作の通知
- OnPassButtonClicked: パス操作の通知
- OnReturnConfirmClicked: パス時カード戻し確定の通知
