## 実装詳細

### 1. Validate(List\<Card\> cards, int targetNumber) 🔵

**信頼性**: 🔵 *interfaces.md ArithmeticValidator、REQ-103より*

2〜3枚のカードで場札数字と一致する計算式が存在するかをboolで返す。

```csharp
public bool Validate(List<Card> cards, int targetNumber)
```

- 2枚の場合: [a,b] の全順列(2通り) × 演算子(+, -) = 4パターン
  - a+b, a-b, b+a, b-a（ただし a+b == b+a なので実質3パターン）
- 3枚の場合: 全順列(6通り) × 演算子パターン(4通り: ++, +-, -+, --) = 24パターン
- 各パターンで左から順に計算し、途中結果がマイナスになったら除外
- 最終結果 >= 0 かつ result % 10 == targetNumber なら valid

**実装ファイル**: `Assets/_Project/Scripts/Core/Systems/ArithmeticValidator.cs`

### 2. FindValidExpressions(List\<Card\> cards, int targetNumber) 🔵

**信頼性**: 🔵 *interfaces.md ArithmeticValidatorより*

全ての有効な演算子の組み合わせを `List<ArithmeticExpression>` で返す。

```csharp
public List<ArithmeticExpression> FindValidExpressions(List<Card> cards, int targetNumber)
```

**実装ファイル**: `Assets/_Project/Scripts/Core/Systems/ArithmeticValidator.cs`

### 3. ArithmeticExpression class 🔵

**信頼性**: 🔵 *interfaces.md データ構造より*

計算式の結果を表現するデータクラス。

```csharp
public class ArithmeticExpression
{
    public List<Card> Cards { get; }       // 使用カード（順序付き）
    public List<char> Operators { get; }   // 演算子リスト ('+' or '-')
    public int Result { get; }             // 計算結果
    public string Expression { get; }      // "7 - 5 + 1 = 3" 形式の文字列
}
```

**実装ファイル**: `Assets/_Project/Scripts/Core/Systems/ArithmeticValidator.cs`

## 単体テスト要件

### 1. 2枚足し算: [3,4] target=7 → valid 🔵 (EDGE-201)

3+4=7 で場札7に一致することを確認する。

### 2. 2枚引き算: [9,2] target=7 → valid (9-2=7) 🔵 (EDGE-202)

9-2=7 で場札7に一致することを確認する。

### 3. 2枚引き算(逆順): [2,9] target=7 → valid (9-2=7, 順列考慮) 🔵

入力順が[2,9]でも順列を考慮して9-2=7が見つかることを確認する。

### 4. 2枚マイナス禁止: [2,5] target=3 → 5-2=3 valid, 2-5=-3 invalid 🔵 (REQ-104)

5-2=3は有効だが、2-5=-3は計算過程でマイナスになるため除外されることを確認する。

### 5. 3枚混合: [7,5,1] target=3 → 7-5+1=3 valid 🔵 (EDGE-203)

3枚の加減混合計算で場札に一致することを確認する。

### 6. 3枚マイナス禁止: [5,1,7] target=3 → 5+1-7=-1 invalid 🔵 (EDGE-203)

5+1-7=-1 のように計算途中でマイナスにならなくても最終結果がマイナスになる式が除外されることを確認する。ただし他の順列で有効な式が存在する場合はValidateはtrueを返す。

### 7. 計算途中の0許容: [5,5] target=0 → 5-5=0 valid 🔵 (EDGE-204)

5-5=0 で計算結果が0でも有効であることを確認する。

### 8. 一の位判定: [8,9] target=7 → 8+9=17, 17%10=7 valid 🔵

8+9=17 の一の位7が場札7に一致することを確認する。

### 9. FindValidExpressions: 複数の有効式がある場合すべて返す 🔵

有効な計算式が複数存在する場合、FindValidExpressionsが全ての有効式をリストで返すことを確認する。

**テストファイル**: `Assets/_Project/Tests/EditMode/Core/Systems/ArithmeticValidatorTests.cs`
