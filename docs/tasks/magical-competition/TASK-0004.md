カードの静的データを保持するScriptableObjectの定義と、45枚の魔石カードアセットを自動生成するエディタツールの作成を行う。

## 実装詳細

### 1. CardData ScriptableObject 🔵

**信頼性**: 🔵 *interfaces.md CardDataより*

```csharp
[CreateAssetMenu(fileName = "NewCardData", menuName = "MagicalCompetition/CardData")]
public class CardData : ScriptableObject
{
    public CardColor color;
    public int number;        // 1~9
    public Sprite cardSprite; // カード表面
    public Sprite backSprite; // カード裏面（共通）
}
```

**実装ファイル**: `Assets/_Project/Scripts/Core/Model/CardData.cs`

### 2. CardDataGenerator エディタツール 🔵

**信頼性**: 🔵 *architecture.md Editor拡張より*

MenuItemから45枚の魔石カードアセットを一括生成するエディタツール。

- note.mdのファイルプレフィックスマッピングに基づきスプライト自動割り当て:

| 色 | プレフィックス範囲 |
|---|---|
| Fire | 030-038 |
| Water | 010-018 |
| Light | 020-028 |
| Earth | 040-048 |
| Wind | 050-058 |

- 裏面スプライト(002)は全カード共通
- 出力先: `Assets/_Project/Data/Cards/`
- ファイル命名: `{Color}_{Number}.asset`（例: `Fire_1.asset`）

**実装ファイル**: `Assets/_Project/Scripts/Editor/CardDataGenerator.cs`

## 実装手順

### DIRECT手順

1. **direct-setup**: CardData.csをCore/Model/に作成する
2. **direct-setup**: CardDataGenerator.csをEditor/に作成する
3. **direct-setup**: スプライトマッピングテーブルをGeneratorに定義する
4. **direct-verify**: Unityエディタでコンパイルエラーがないことを確認する
5. **direct-verify**: メニューから生成を実行し、45枚のアセットが正しく生成されることを確認する
6. **direct-verify**: 各アセットのInspectorで色・番号・スプライトが正しいことを確認する
