### 2. カード選択UI 🔵

**信頼性**: 🔵 *NFR-101より*

タップ/クリックで選択状態を切り替える。複数選択が可能（同数字・同色・計算出し用）。

```csharp
public event Action<CardView> OnCardClicked;

public List<CardView> GetSelectedCards()
{
    // 選択中のCardViewリストを返す
}

    // 全選択を解除する
```

- 各CardViewにクリックイベントリスナーを設定
- 選択状態はCardView.SetSelected()で視覚フィードバック

### 3. グレーアウト制御 🔵

**信頼性**: 🔵 *NFR-102より*

出せるカードと出せないカードを視覚的に区別する。

```csharp
public void UpdateCardStates(List<PlayAction> validPlays)
{
    // validPlaysに含まれるカードは通常表示
    // 含まれないカードはSetInteractable(false)でグレーアウト
}
```

- PlayActionリストからプレイ可能なカードを抽出
- 該当しないCardViewにSetInteractable(false)を適用

### 4. アニメーション 🔵

**信頼性**: 🔵 *architecture.md Viewsより*

カード配布時・出し時のアニメーション。

- カード配布: 山札位置から手札位置への移動アニメーション
- カード出し: 手札位置から場札位置への移動アニメーション
- CardView.PlayMoveAnimation()を利用

## PlayModeテスト要件

### 1. 手札表示: CardViewが手札枚数分生成される 🔵

UpdateHandを呼び出した際、手札の枚数と同じ数のCardViewが生成されることを確認する。

### 2. カード選択: タップで選択状態が切り替わる 🔵

CardViewをクリックした際、選択状態がトグルされ、OnCardClickedイベントが発火することを確認する。

### 3. 複数選択: 複数カードが同時に選択できる 🔵

複数のCardViewを順にクリックした際、全てが選択状態になり、GetSelectedCardsが全てを返すことを確認する。

### 4. グレーアウト: 出せないカードがグレーアウトされる 🔵

UpdateCardStatesを呼び出した際、有効手に含まれないカードがグレーアウト表示になることを確認する。

### 5. カード追加: 新しいCardViewが追加される 🔵

UpdateHandでカード枚数が増えた際、新しいCardViewが追加されることを確認する。

### 6. カード削除: 対応するCardViewが削除される 🔵

UpdateHandでカード枚数が減った際、対応するCardViewが削除されることを確認する。
