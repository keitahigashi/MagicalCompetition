手札補充処理を実装する。ゲーム開始時の初期ドロー（各プレイヤー3枚）と、ターンごとの手札補充（3枚まで）を行う。山札が不足した場合は残りを全て引き、山札が空になった場合はリーチ状態に移行する。

## 実装詳細

### 1. InitialDraw(List\<PlayerState\> players) 🔵

**信頼性**: 🔵 *dataflow.md Setup フェーズ、REQ-004より*

ゲーム開始時に各プレイヤーが山札から3枚引いて手札にする。

```csharp
public void InitialDraw(List<PlayerState> players)
```

- 各プレイヤーに対してDrawFromDeckを3回呼び出し、手札に追加
- プレイヤーの山札はCardDistributorで事前に設定されている前提

**実装ファイル**: `Assets/_Project/Scripts/Core/Systems/DrawSystem.cs`

### 2. Refill(PlayerState player) 🔵

**信頼性**: 🔵 *dataflow.md Draw フェーズ、REQ-108, REQ-109より*

手札が3枚になるまで山札から補充する。山札不足・空の場合の処理も含む。

```csharp
public void Refill(PlayerState player)
```

処理フロー:
1. `player.IsReach == true` の場合 → 何もしない（既にリーチ）
2. 手札が3枚以上の場合 → 何もしない
3. 手札が3枚未満の場合:
   - 山札十分: 3枚になるまで `DrawFromDeck()` で補充
   - 山札不足: 残り全てを引く（3枚未満でも終了）
   - 山札が空になった: `player.IsReach = true` に設定

**実装ファイル**: `Assets/_Project/Scripts/Core/Systems/DrawSystem.cs`

## 単体テスト要件

### 1. InitialDraw: 各プレイヤーの手札が3枚になる 🔵

InitialDraw実行後に全プレイヤーの手札が3枚であることを確認する。

### 2. InitialDraw: 山札から3枚減る 🔵

InitialDraw実行後に各プレイヤーの山札が3枚減っていることを確認する。

### 3. Refill(手札0枚): 3枚引く 🔵

手札が空の状態からRefillを実行し、3枚が手札に追加されることを確認する。

### 4. Refill(手札1枚): 2枚引く 🔵

手札が1枚の状態からRefillを実行し、2枚が追加されて合計3枚になることを確認する。

### 5. Refill(手札3枚): 引かない 🔵

手札が既に3枚ある状態でRefillを実行し、手札が変化しないことを確認する。

### 6. Refill(山札不足): 残り全部引いて終了 🔵

山札が2枚しかない状態で手札0枚からRefillを実行し、2枚だけ引いて手札が2枚になることを確認する。

### 7. Refill(山札空になった): IsReach=trueに設定 🔵 (REQ-109)

Refill実行後に山札が空になった場合、IsReachがtrueに設定されることを確認する。

### 8. Refill(既にリーチ): 何もしない 🔵

IsReach=trueの状態でRefillを実行し、手札・山札に変化がないことを確認する。

**テストファイル**: `Assets/_Project/Tests/EditMode/Core/Systems/DrawSystemTests.cs`
