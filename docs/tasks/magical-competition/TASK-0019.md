## 実装詳細

### State Machine遷移図

```
Setup → SelectFieldCard → Play → Draw → CheckWin → NextTurn → Play...
              ↑                                        │
              └──── AllPassReset ←─────────────────────┘
                                                 End
```

### 1. GameStateMachine class 🔵

**信頼性**: 🔵 *architecture.md State Machineより*

ゲームフェーズの状態遷移を管理するクラス。

```csharp
public class GameStateMachine
{
    public GamePhase CurrentPhase { get; }
    public void TransitionTo(GamePhase next);
    public event Action<GamePhase> OnPhaseEnter;
    public event Action<GamePhase> OnPhaseExit;
}
```

- `CurrentPhase`: 現在のゲームフェーズを返すプロパティ
- `TransitionTo(GamePhase next)`: 次のフェーズへ遷移する。不正な遷移の場合はInvalidOperationExceptionをスローする
- `OnPhaseEnter`: フェーズ開始時に発火するイベント（新しいフェーズを引数に渡す）
- `OnPhaseExit`: フェーズ終了時に発火するイベント（終了するフェーズを引数に渡す）

**実装ファイル**: `Assets/_Project/Scripts/Controllers/GameStateMachine.cs`

### 2. 遷移ルール 🔵

**信頼性**: 🔵 *architecture.md 遷移ルールより*

以下の遷移のみを許可する。それ以外の遷移はエラーとする。

| 現在のフェーズ | 許可される遷移先 |
|---|---|
| Setup | SelectFieldCard |
| SelectFieldCard | Play |
| Play | Draw（カード出し時） |
| Play | NextTurn（パス時、全員パスでない場合） |
| Play | AllPassReset（全員パス時） |
| Draw | CheckWin |
| CheckWin | NextTurn（継続） |
| CheckWin | End（勝利） |
| NextTurn | Play |
| AllPassReset | SelectFieldCard |

**実装ファイル**: `Assets/_Project/Scripts/Controllers/GameStateMachine.cs`

## 単体テスト要件

### 1. 初期フェーズ: Setup 🔵

GameStateMachine生成直後のCurrentPhaseがSetupであることを確認する。

### 2. 正常遷移: Setup→SelectFieldCard→Play→Draw→CheckWin→NextTurn 🔵

基本的なゲームフローの遷移が正常に完了することを確認する。

### 3. パス遷移: Play→NextTurn (全員パスでない) 🔵

パス時にPlay→NextTurnへの遷移が正常に行われることを確認する。

### 4. 全員パス: Play→AllPassReset→SelectFieldCard 🔵

全員パス時にPlay→AllPassReset→SelectFieldCardの遷移が正常に行われることを確認する。

### 5. 勝利: CheckWin→End 🔵

勝利条件成立時にCheckWin→Endへの遷移が正常に行われることを確認する。

### 6. 不正遷移: Setup→Play → エラー 🔵

許可されていない遷移（Setup→Play）を実行した場合、InvalidOperationExceptionがスローされることを確認する。

### 7. OnPhaseEnter/Exit: イベントが発火する 🔵

フェーズ遷移時にOnPhaseExitイベント（旧フェーズ）とOnPhaseEnterイベント（新フェーズ）が正しい順序で発火することを確認する。

**テストファイル**: `Assets/_Project/Tests/EditMode/Controllers/GameStateMachineTests.cs`
