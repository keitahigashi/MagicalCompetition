## タスク概要

勝利判定と得点計算の2つのクラスを実装する。WinCheckerはプレイヤーの手札と山札が共に0枚かどうかで勝利を判定し、ScoreCalculatorは勝者以外のプレイヤーの残カード合計枚数を得点として計算する。

## 実装詳細

### 1. WinChecker.Check(PlayerState player) 🔵

**信頼性**: 🔵 *dataflow.md CheckWin フェーズ、REQ-110より*

プレイヤーの手札と山札の合計が0枚かどうかで勝利を判定する。

```csharp
public class WinChecker
{
    public bool Check(PlayerState player)
    // player.Hand.Count == 0 && player.Deck.Count == 0
}
```

- PlayerState.HasWonプロパティと同等だが、システムクラスとして独立させる
- ターン終了時にカードプレイ後の状態で判定する

**実装ファイル**: `Assets/_Project/Scripts/Core/Systems/WinChecker.cs`

### 2. ScoreCalculator.Calculate(PlayerState winner, List\<PlayerState\> allPlayers) 🔵

**信頼性**: 🔵 *REQ-202 得点計算ルールより*

勝者以外のプレイヤーの手札+山札の合計枚数を得点として返す。

```csharp
public class ScoreCalculator
{
    public int Calculate(PlayerState winner, List<PlayerState> allPlayers)
    // allPlayers.Where(p => p != winner).Sum(p => p.TotalCardCount)
}
```

- 勝者自身のカードは計算に含めない
- 全ての敗者プレイヤーの手札枚数+山札枚数を合算する

**実装ファイル**: `Assets/_Project/Scripts/Core/Systems/ScoreCalculator.cs`

## 単体テスト要件

### 1. Check(手札0・山札0): true 🔵

手札が0枚かつ山札が0枚の場合にtrueが返されることを確認する。

### 2. Check(手札1・山札0): false 🔵

手札が1枚残っている場合にfalseが返されることを確認する。

### 3. Check(手札0・山札1): false 🔵

手札は0枚だが山札が1枚残っている場合にfalseが返されることを確認する。

### 4. Check(手札3枚出して山札空): true 🔵 (EDGE-103)

手札を3枚全て出し切り、山札も空の状態（リーチ後に手札を全て出した場合）でtrueが返されることを確認する。

### 5. Calculate: 他PL2人の残カード合計 🔵

勝者以外の2人のプレイヤーの残カード（手札+山札）合計枚数が正しく計算されることを確認する。

### 6. Calculate: 勝者のカードは含まれない 🔵

勝者プレイヤーにカードが残っていた場合（通常はないが安全のため）、勝者のカードが得点計算に含まれないことを確認する。

**テストファイル**: `Assets/_Project/Tests/EditMode/Core/Systems/WinCheckerTests.cs`, `Assets/_Project/Tests/EditMode/Core/Systems/ScoreCalculatorTests.cs`
