# Magical Competition 設計ヒアリング記録

**作成日**: 2026-03-13
**ヒアリング実施**: step4 既存情報ベースの差分ヒアリング

## ヒアリング目的

要件定義書・既存設計文書（CLAUDE.md・tech-stack.md）を確認し、技術設計における不明点・設計判断を明確化するためのヒアリングを実施しました。

## 質問と回答

### Q1: ゲームフェーズの遷移フロー

**質問日時**: 2026-03-13
**カテゴリ**: アーキテクチャ
**背景**: CLAUDE.mdでは `Setup → Draw → Play → Judge → NextTurn → End` の6フェーズが定義されていたが、要件定義書の内容（初期場札設定・全員パスリセット・勝利判定）を反映すると8フェーズが必要と判断。

**提案したフロー**:
```
Setup → SelectFieldCard → Play → Draw → CheckWin → NextTurn
                ↑                                        │
                └──── AllPassReset ←─────────────────────┘
                                                   End
```

**回答**: 問題ない

**信頼性への影響**:
- GamePhase enum の設計が 🟡 → 🔵 に向上
- State Machine の遷移図が確定

---

### Q2: カードプレイのUI操作方式

**質問日時**: 2026-03-13
**カテゴリ**: UI/UX設計
**背景**: カードの出し方について、ドラッグ＆ドロップ方式とタップ選択→確定ボタン方式の2つが考えられた。複数枚同時出し（同数字・同色・計算出し）の操作性を考慮する必要があった。

**回答**: 選択→確定（カードをタップして選択状態にし、「出す」ボタンで確定）

**信頼性への影響**:
- PlayerInputController の設計が 🟡 → 🔵 に向上
- HandView のインタラクション設計が確定
- ドラッグ＆ドロップ関連の実装が不要と確定

---

### Q3: アニメーションの複雑度

**質問日時**: 2026-03-13
**カテゴリ**: 技術選択
**背景**: カードアニメーションの複雑度により実装方針が変わる。高度なアニメーション（カードフリップ・パーティクル等）が必要ならDOTweenが必須だが、基本的な移動・フェードであればCoroutineでも対応可能。

**回答**: 基本的（移動＋フェード、DOTween使用OK）

**信頼性への影響**:
- CardView のアニメーション設計が 🟡 → 🔵 に向上
- DOTween（またはCoroutine）による実装方針が確定
- 複雑なパーティクル・3D演出は不要と確定

---

### Q4: 画面レイアウト構成

**質問日時**: 2026-03-13
**カテゴリ**: UI/UX設計
**背景**: ゲーム画面のレイアウトについて、上部にAI情報、中央に場札、下部にプレイヤー手札＋操作ボタンという構成を提案。

**提案したレイアウト**:
```
┌───────────────────────────────────────────┐
│          AI情報エリア（上部）               │
│  [AI1: 手札3 山札8] [AI2: 手札2 山札5 ⚡]  │
├───────────────────────────────────────────┤
│          場札エリア（中央）                 │
│        ┌─────────┐                        │
│        │  火 ・ 5 │                        │
│        └─────────┘                        │
│     ターン表示: 「あなたのターン」           │
├───────────────────────────────────────────┤
│  プレイヤーエリア（下部）                   │
│  ┌───┐ ┌───┐ ┌───┐    ┌────┐             │
│  │ C1│ │ C2│ │ C3│    │山札│             │
│  └───┘ └───┘ └───┘    │ 12 │             │
│   手札（横並び）        └────┘             │
│      [出す]  [パス]                        │
└───────────────────────────────────────────┘
```

**回答**: 問題ない

**信頼性への影響**:
- 全View系クラスのレイアウト設計が 🟡 → 🔵 に向上
- AIInfoView, FieldView, HandView, DeckView, TurnIndicatorView の配置が確定

## ヒアリング結果サマリー

### 確認できた事項
- ゲームフェーズは8フェーズ構成（Setup, SelectFieldCard, Play, Draw, CheckWin, NextTurn, AllPassReset, End）
- UI操作は「選択→確定」方式（ドラッグ＆ドロップ不要）
- アニメーションは基本レベル（移動＋フェード）、DOTween使用可
- 画面レイアウトは上部AI情報・中央場札・下部プレイヤーの3段構成

### 設計方針の決定事項
- State Machine は8フェーズで設計
- PlayerInputController はタップ選択＋確定ボタン方式
- CardView のアニメーションは DOTween または Coroutine で実装
- Canvas レイアウトは VerticalLayoutGroup ベースの3段構成

### 残課題
- なし（全件解消済み）
